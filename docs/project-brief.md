# 项目总纲

## 目标

为企业内部员工构建 RAG 知识库系统，支持自然语言问答电子产品手册/维修指南，秒级返回答案。

## 目标用户

- 企业内部员工，非技术人员
- 几十人并发

## 核心问题

员工翻 PDF 手册找信息低效。系统让人用自然语言提问，检索相关片段，LLM 生成回答。

## 数据来源

- 格式：PDF + Markdown
- 语言：中文
- 量级：几十本手册，单本几十到几百页
- 含图片

## 部署

- Docker + NVIDIA GPU
- 内网可跑本地模型
- 不需要隐私保护

## 约束

- 几周交付 MVP
- 中文 PDF 双栏/表格排版复杂，分块质量定天花板
- Milvus 运维比 Qdrant 重，功能更强
- MCP 网络搜索 + HyDE 增加延迟和故障点

## 方法论: TDD

红 → 绿循环，一 slice 一次。先写失败测试，再写刚好能过的代码。

分层测试：
- **单元测试**：纯函数、工具函数、分块/解析/重排序逻辑，无外部依赖，毫秒级
- **集成测试**：FastAPI router、LangGraph 节点、LangChain 调用链，**mock 外部 LLM/Milvus**，CI 独立可跑
- **E2E 测试**（可选，手动触发）：真实 LLM API，不跑在 CI，单独脚本

测试只测 seam（公共接口），不耦合实现细节。Mock 只 mock 外部服务边界，不 mock 内部 collaborator。

## 成功标准

- 回答准确率（人工抽查询问核对）
- 响应速度（几秒内）
- 多轮对话可用
- P0 功能集成测试覆盖率 ≥ 80%

## 反目标

- 不做页码溯源
- 不做多语言
- 不做语音
- 不做文档自动同步更新
- 不做权限隔离
